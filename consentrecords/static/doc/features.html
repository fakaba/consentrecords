Stakeholder UIs

    Workflows

    4. Sign Out

  2. organization: https://www.consentrecords.org
  3. administration: https://www.consentrecords.org/list

Data Organization

  1. Instances of specified types organized hierarchically.

    For example: Organization -> Site -> Offering -> Session -> Period

  2. Instances have associated values of various data types

Configuration

Configuration for the data.

Data types

  1. Reference
  2. Text
  3. Translation
  4. Number
  5. Datestamp
  6. Month
  7. Datestamp (day optional)
  8. Time
  9. email
  10. url
  11. telephone

Enumerations

  1. Configurable via Terms

UI Langauges

  1. English

Data auditing

  1. Instances and values are write-once.
  2. Instances and values are deleted by writing a record stating when they are deleted.
  3. All writes of instances and values are associated with a transaction that records the user and the timestamp

Value Ordering

  1. Values have a 0-based position as part of their record.
  2. Positions are written at the time the value is written.
  3. No UI for changing positions.

API

  1. JSON apis to the middle tier.
  2. Operations to get data without user login are available externally, subject to security constraints.
