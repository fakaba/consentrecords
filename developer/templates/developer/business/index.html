Business Layer Overview

This page provides an overview of the business layer of the BC&YI platform.

The business layer is implemented as a Python/Django web server running on Amazon Web Services. The business layer is accessed from client systems and other systems via https, a protocol for secure communication over a computer network which is widely used on the Internet.

The managed in the business layer are not domain specific. Instead, they represent a set of typed objects that are connected in a hierarchy as well as a set of links between objects in the system. The types of the objects are determined by configuration, which is stored in the database just like other objects. For example, the configuration includes users, organizations and offerings of those organizations.

Each object in the system has one or more sets of values that are associated with an object. For example, a user has email addresses, first names and last names. The specification of what values are associated with an object are determined by configuration as well.

Each object and each value is represented by a Universial Unique Identifier (UUID) based on version 4 of RFC 4122. The UUIDs are represented as a hexidecimal 32-digit string.

The business layer provides an Application Programming Interface (API) which other systems can use to interact with the objects represented in the business layer.

Objects in the system are identified using path string in a format similar to CSS, a standard from the world-wide-web consortium. The paths supported in this system are a subset of the CSS specification.
