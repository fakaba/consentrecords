Web API Calls

The BC&YI system provides an Application Programming Interface (API) which other systems can use to interact with the objects represented in the business layer. This API is accessed over the Internet using the https: protocol. As such, it can be implemented on web pages, on other web servers or any program that accesses the library.

There are two methods of the HTTPS protocol that are used in the API: GET and POST. Each API call also has additional parameters that are included using the standard scheme for GET and POST methods. In the following example, a GET method (which can be called using a browser), is used to get information about all of the Service objects in the system:

https://www.consentrecords.org/api/selectall/?path=Service

Each of the APIs returns its data in json format. The API reference describes the keys for each field in data returned.
