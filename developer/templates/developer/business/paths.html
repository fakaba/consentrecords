Path Definitions

Objects in the system are identified using path string in a format similar to CSS, a standard from the world-wide-web consortium. The paths supported in this system are a subset of the CSS specification. The CSS specification includes ways to identify single objects or groups of objects based on common characteristics.

Paths can be relatively simple or complex, and objects can be identified by name or by id. In general, if a path is being constructed, using an id is more robust and is recommended.

In its simplest form, a path an consist of a single ID, which is identified by by a hash symbol (#) followed by a 32-digit hexidecimal character. This represents a unique object in the system.

Objects can be identified by type, either using a name or an ID for the type.

Objects can be identified as descendents of other objects, by following a path specification with a right chevron (>) followed by the name or id of a field within objects of that type. For example, one can identify an offering and get a list of the services associated with that offering.

https://www.consentrecords.org/api/selectall/?path=Service
